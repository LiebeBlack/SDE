Acerca del Proyecto

Una solución vanguardista diseñada para revolucionar la gestión documental en instituciones educativas.

Objetivo Principal

Nacido como un proyecto de tesis universitaria, este sistema representa el estado del arte en la administración de expedientes docentes y administrativos. Nuestro enfoque prioriza la eficiencia, la seguridad de la información y una experiencia de usuario fluida y moderna.

Arquitectura Robusta

Implementado mediante un workspace multicrate en Rust, el sistema sigue estrictamente los principios de Clean Architecture (Arquitectura Hexagonal). Esto asegura un código altamente mantenible, fácilmente testeable y preparado para escalar.

Stack Tecnológico

Construido 100% en Rust para garantizar cero fugas de memoria y respuestas en milisegundos. Utiliza SQLite como motor de base de datos embebido para su funcionamiento sin conexión (Offline-First), Axum para su API RESTful ultrarrápida, y un frontend elegante sin frameworks pesados.

Características

Funcionalidades de nivel empresarial diseñadas para instituciones educativas modernas

Seguridad de Grado Militar

Autenticación mediante JWT, control de acceso basado en roles (RBAC) granular, encriptación AES-256 y un registro de auditoría inmutable para cada acción.

Rendimiento Extremo

Aprovechando la potencia de Rust, el servidor procesa y responde peticiones en apenas unos milisegundos, optimizando al máximo los recursos de hardware.

Clean Architecture

Estructura de código modular y escalable que aísla completamente la lógica de negocio de la base de datos y la interfaz, facilitando su evolución continua.

Filosofía Offline-First

Capacidad operativa total incluso sin conexión a internet. La base de datos embebida SQLite permite sincronización diferida y autonomía completa.

Interfaz Premium

Diseño basado en Glassmorphism con animaciones fluidas, totalmente adaptable a móviles y accesible según las estrictas normativas WCAG 2.1 AA.

Panel Inteligente

Dashboard de control centralizado con estadísticas instantáneas, filtros de búsqueda avanzada y visualización de datos dinámica e interactiva.

Detalles de Programación

Arquitectura técnica y stack de tecnologías utilizadas en el desarrollo

Stack Tecnológico

Lenguaje Principal: Rust 1.75+ - Sistema de tipos fuerte, seguridad de memoria y concurrencia sin data races.

Framework Web: Axum - Framework asíncrono moderno basado en Tokio para construir APIs RESTful de alto rendimiento.

Base de Datos: SQLite - Base de datos embebida, portátil y sin servidor, ideal para modo offline-first.

ORM: SQLx - Biblioteca async para SQL con verificación de queries en tiempo de compilación.

Patrones de Diseño

Clean Architecture: Separación de dominio, infraestructura y presentación.

Repository Pattern: Abstracción del acceso a datos para facilitar testing y cambios de implementación.

Service Layer: Lógica de negocio separada de los handlers HTTP.

Dependency Injection: Inyección de dependencias para loose coupling.

Seguridad

Autenticación: JWT (JSON Web Tokens) con claims personalizados.

Autorización: RBAC (Role-Based Access Control) con roles granulares.

Encriptación: Argon2 para passwords, SHA-256 para integridad de archivos.

Validación: Validación de tipos en tiempo de compilación y runtime.

// Ejemplo de Handler con Autenticación
pub async fn crear_expediente(
    State(state): State<AppState>,
    Json(data): Json<CrearExpediente>,
    usuario: Usuario,  // Extraído por middleware JWT
) -> Result<Json<ExpedienteDocente>, AppError> {
    // Verificar permisos RBAC
    if !usuario.tiene_permiso("crear_expediente") {
        return Err(AppError::Forbidden);
    }
    
    // Validar datos
    let expediente = state.expediente_service
        .crear(data, usuario.id).await?;
    
    // Auditar acción
    state.audit_service.registrar(
        usuario.id,
        "CREACION_EXPEDIENTE",
        expediente.id
    ).await;
    
    Ok(Json(expediente))
}

Instalación y Configuración

Guía paso a paso para instalar y configurar el Sistema de Gestión Escolar

Requisitos Previos

Rust: Versión 1.75 o superior instalada

Sistema Operativo: Windows, Linux o macOS

Memoria: Mínimo 512MB RAM

Disco: 100MB para aplicación + espacio para documentos

Instalación Rápida

1. Clonar el repositorio:

git clone https://github.com/LiebeBlack/SDE.git

2. Navegar al directorio:

cd TesisYoangel

3. Compilar en modo release:

cargo build --release

4. Ejecutar el servidor:

./target/release/escuela_api

Configuración

El sistema puede configurarse mediante variables de entorno:

DATABASE_PATH: Ruta de la base de datos SQLite (default: escuela.db)

STORAGE_PATH: Directorio de almacenamiento (default: storage)

BIND_ADDRESS: Dirección del servidor (default: 0.0.0.0:3000)

STATIC_PATH: Ruta de archivos estáticos (default: static)

Guía de Despliegue 
# Variables de Entorno (Linux/macOS)
export DATABASE_PATH=mi_escuela.db
export STORAGE_PATH=archivos
export BIND_ADDRESS=0.0.0.0:3000
export STATIC_PATH=static

# Variables de Entorno (Windows)
set DATABASE_PATH=mi_escuela.db
set STORAGE_PATH=archivos
set BIND_ADDRESS=0.0.0.0:3000
set STATIC_PATH=static

# Ejecutar servidor
cargo run --release

Arquitectura Enterprise

Nuestro sistema sigue los principios de Clean Architecture con separación clara de responsabilidades. Cada capa tiene una función específica y bien definida, asegurando mantenibilidad y escalabilidad a largo plazo.

La arquitectura multicrate de Rust permite compilación independiente, testing granular, y fácil integración de nuevas funcionalidades sin afectar el código existente.

Documentación Completa
